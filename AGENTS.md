# Agent 协作规则

## 安装

- 用户要求安装本项目时，先完整读取 `INSTALL.md` 和 `README.md`。
- 严格执行环境检查、初始化、验证和启动步骤，不得把“页面能打开”当作安装完成。
- 初始化配置必须逐项向用户确认；已有数据时不得自动覆盖。

## 数据操作

- 所有用户数据以本机数据目录中的 `state.json` 为权威源。
- 日常操作必须优先使用 `npm run cockpit -- ...`，不要直接修改 `state.json`。
- “已发布”必须通过 `publish:add` 生成，并保留链接、作品 ID 或证据说明之一。
- “已复盘”必须通过 `review:add --confirm` 生成。
- 修改后运行 `npm run cockpit -- status` 复核结果。

## 开发

- 保持服务只监听 `127.0.0.1`。
- 使用跨平台 Node.js API，不新增只支持 Bash、PowerShell 或 macOS 的核心安装逻辑。
- 手工代码修改使用小范围、可审查的变更。
- 完成修改后运行 `npm run verify`。
- 不提交本地配置、用户数据、截图、凭证、绝对路径、`node_modules` 或构建产物。

## 产品边界

- 这是展示和维护内容工作流的本地看板，不在页面内嵌 AI 聊天入口。
- Codex、Claude Code 等本地 AI 通过 CLI 与看板协作。
- 不自动发布内容，不自动抓取平台账号，不替用户确认复盘结论。
